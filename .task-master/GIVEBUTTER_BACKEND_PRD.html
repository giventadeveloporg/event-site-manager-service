1. Overview

This PRD details the backend implementation of Givebutter integration for zero-fee fundraising and Mass offerings. The implementation follows the existing Stripe payment adapter pattern, ensuring consistency and maintainability.

✅ Architecture Pattern: Multi-tenant, database-driven configuration matching Stripe implementation. All credentials stored encrypted in payment_provider_config table, retrieved dynamically at runtime.

2.2 Payment Provider Config Structure

The existing payment_provider_config table supports Givebutter configuration:

  • config_data: Encrypted JSON containing API key: {"api_key": "gb_live_xxx"}
  • webhook_secret: Encrypted Givebutter webhook secret
  • metadata: JSONB containing Givebutter-specific config (campaign IDs, use case, preset amounts, etc.)

3. Core Components

3.1 GivebutterPaymentAdapter

Purpose: Implements PaymentAdapter interface for Givebutter payment processing.

Key Responsibilities:

  • Retrieve tenant's Givebutter configuration from database
  • Decrypt API key and webhook secret
  • Call Givebutter API to create donations
  • Handle Givebutter API responses
  • Return payment session with checkout URL

Implementation Pattern:

@Service
@RequiredArgsConstructor
@Slf4j
public class GivebutterPaymentAdapter implements PaymentAdapter {

    private final PaymentProviderConfigRepository configRepository;
    private final EncryptionService encryptionService;
    private final RestTemplate restTemplate;

    @Override
    public PaymentSession initializePayment(PaymentInitializeRequest request) {
        // 1. Resolve tenant's Givebutter configuration dynamically
        PaymentProviderConfig config = resolveProviderConfig(
            request.getTenantId(),
            request.getPaymentType()  // OFFERING or DONATION
        );

        // 2. Decrypt API key from encrypted config_data
        String apiKey = decryptApiKey(config);

        // 3. Extract Givebutter-specific config from metadata
        Map<String, Object> metadata = config.getMetadata();
        String campaignId = (String) metadata.get("campaignId");

        // 4. Build Givebutter donation request
        GivebutterDonationRequest gbRequest = buildDonationRequest(request, campaignId);

        // 5. Call Givebutter API with tenant-specific credentials
        HttpHeaders headers = new HttpHeaders();
        headers.set("Authorization", "Bearer " + apiKey);
        headers.setContentType(MediaType.APPLICATION_JSON);

        ResponseEntity<GivebutterDonationResponse> response =
            restTemplate.postForEntity(
                "https://api.givebutter.com/v1/donations",
                new HttpEntity<>(gbRequest, headers),
                GivebutterDonationResponse.class
            );

        // 6. Return payment session
        return PaymentSession.builder()
            .providerType(ProviderType.GIVEBUTTER)
            .providerTransactionId(response.getBody().getDonationId())
            .sessionUrl(response.getBody().getCheckoutUrl())
            .build();
    }

    private PaymentProviderConfig resolveProviderConfig(
        String tenantId,
        PaymentType paymentType
    ) {
        // Determine use case based on payment type
        String useCase = null;
        if (PaymentType.OFFERING.equals(paymentType)) {
            useCase = "OFFERING";
        } else if (PaymentType.DONATION.equals(paymentType)) {
            useCase = "DONATION_ZERO_FEE";
        }

        // Query for Givebutter config matching tenant and use case
        List<PaymentProviderConfig> configs = configRepository
            .findByTenantIdAndProviderType(tenantId, ProviderType.GIVEBUTTER);

        if (useCase != null) {
            return configs.stream()
                .filter(c -> {
                    Map<String, Object> meta = c.getMetadata();
                    return useCase.equals(meta.get("useCase"));
                })
                .findFirst()
                .orElseThrow(() -> new PaymentException(
                    "Givebutter not configured for tenant: " + tenantId
                ));
        }

        return configs.stream()
            .filter(PaymentProviderConfig::getIsActive)
            .findFirst()
            .orElseThrow(() -> new PaymentException(
                "Givebutter not configured for tenant: " + tenantId
            ));
    }
}

3.2 PaymentOrchestrationService Updates

Purpose: Route payment requests to appropriate provider adapter based on payment type and event configuration.

Routing Logic:

private ProviderType determineProvider(PaymentInitializeRequest request) {
    // For OFFERING payment type, prefer Givebutter if configured
    if (PaymentType.OFFERING.equals(request.getPaymentType())) {
        boolean hasGivebutter = configRepository
            .existsByTenantIdAndProviderType(
                request.getTenantId(),
                ProviderType.GIVEBUTTER
            );

        if (hasGivebutter) {
            return ProviderType.GIVEBUTTER;
        }
    }

    // For DONATION payment type, check if zero-fee provider is configured
    if (PaymentType.DONATION.equals(request.getPaymentType())) {
        // Check event metadata for zero-fee configuration
        if (request.getEventId() != null) {
            EventDetails event = eventRepository.findById(request.getEventId())
                .orElse(null);

            if (event != null) {
                Map<String, Object> metadata = event.getMetadata();
                if (metadata != null) {
                    Map<String, Object> donationConfig =
                        (Map<String, Object>) metadata.get("donationConfig");

                    if (donationConfig != null) {
                        Boolean useZeroFeeProvider =
                            (Boolean) donationConfig.get("useZeroFeeProvider");

                        if (Boolean.TRUE.equals(useZeroFeeProvider)) {
                            String zeroFeeProvider =
                                (String) donationConfig.get("zeroFeeProvider");

                            if ("GIVEBUTTER".equals(zeroFeeProvider)) {
                                boolean hasGivebutter = configRepository
                                    .existsByTenantIdAndProviderType(
                                        request.getTenantId(),
                                        ProviderType.GIVEBUTTER
                                    );

                                if (hasGivebutter) {
                                    return ProviderType.GIVEBUTTER;
                                }
                            }
                        }
                    }
                }
            }
        }
    }

    // Default to Stripe (primary provider for tickets)
    return ProviderType.STRIPE;
}

3.3 GivebutterWebhookController

Purpose: Handle webhook events from Givebutter for payment status updates.

Endpoint:

@RestController
@RequestMapping("/api/webhooks/givebutter")
@RequiredArgsConstructor
@Slf4j
public class GivebutterWebhookController {

    private final PaymentProviderConfigRepository configRepository;
    private final EncryptionService encryptionService;
    private final PaymentOrchestrationService orchestrationService;

    @PostMapping
    public ResponseEntity<?> handleWebhook(
        @RequestHeader(value = "X-Givebutter-Signature", required = false) String signature,
        @RequestBody String payload,
        HttpServletRequest request
    ) {
        // 1. Parse webhook payload to extract tenant_id
        GivebutterWebhookEvent event = objectMapper.readValue(
            payload,
            GivebutterWebhookEvent.class
        );

        // 2. Extract tenant_id from donation metadata or campaign
        String tenantId = extractTenantId(event);

        // 3. Retrieve webhook secret from database (per tenant)
        PaymentProviderConfig config = configRepository
            .findByTenantIdAndProviderType(tenantId, ProviderType.GIVEBUTTER)
            .orElse(null);

        // 4. Decrypt webhook secret
        String webhookSecret = decryptWebhookSecret(config);

        // 5. Verify webhook signature
        if (!verifySignature(payload, signature, webhookSecret)) {
            return ResponseEntity.status(401).body("Invalid signature");
        }

        // 6. Process webhook event
        orchestrationService.processGivebutterWebhook(event, tenantId);

        return ResponseEntity.ok().build();
    }
}

Webhook Events Supported:

  • donation.completed - Donation succeeded
  • donation.updated - Donation updated
  • donation.refunded - Donation refunded

7. Security Considerations

7.1 Credential Storage

  • API keys encrypted using AES-256-GCM before storage
  • Webhook secrets encrypted separately
  • No credentials in application.properties or environment variables
  • Per-tenant encryption keys (if supported by EncryptionService)

7.2 Webhook Security

  • Verify HMAC-SHA256 signature for all webhook requests
  • Retrieve webhook secret dynamically by tenant_id
  • Log all webhook events for audit trail
  • Idempotent webhook processing (check for duplicate events)

7.3 API Security

  • API keys never exposed to frontend
  • All Givebutter API calls made from backend only
  • Rate limiting on webhook endpoints
  • Input validation on all DTOs
