Promotion Email Management System - Backend PRD

📋 Project Context: This project is already in active development phase. All infrastructure, basic development tools, database schema foundation, and JSON schema definitions are already in place. This PRD focuses exclusively on feature implementation and does not include any infrastructure setup, basic tooling, or foundational development work.

1. Overview

This document outlines the backend implementation requirements for the Promotion Email Management System. The system will enable administrators to create, manage, and send promotion email templates associated with events and discount codes.

2. Objectives

  • Implement database-backed promotion email template storage
  • Create REST API endpoints for template CRUD operations
  • Implement email sending functionality (test and bulk)
  • Create audit logging for all sent emails
  • Integrate with existing event and discount code systems

4.2 Service Classes

Task 5.1: Create PromotionEmailTemplateService

Priority: HIGH

Description: Implement service layer for template operations

Required Methods:

  • createTemplate(formDTO, tenantId, userId) - Create new template
  • updateTemplate(id, formDTO, tenantId) - Update existing template
  • getTemplate(id, tenantId) - Get template by ID
  • listTemplates(criteria, tenantId) - List templates with filtering
  • deleteTemplate(id, tenantId) - Soft delete (set isActive=false)
  • validateTemplateName(name, eventId, tenantId, excludeId) - Check uniqueness

Business Rules:

  • Template name must be unique per event per tenant
  • Event must exist and belong to tenant
  • Discount code (if provided) must exist and belong to event
  • Auto-set createdById and updatedAt timestamps

Task 5.2: Create PromotionEmailService

Priority: HIGH

Description: Implement email sending service

Required Methods:

  • sendTestEmail(templateId, recipientEmail, tenantId, userId) - Send test email
  • sendBulkEmail(templateId, recipientEmails, tenantId, userId) - Send bulk emails
  • buildEmailContent(template, overrides) - Build final email content
  • logEmailSent(logEntry) - Log sent email

Integration Points:

  • Use existing email service infrastructure
  • Retrieve event registrants from EventAttendee/EventTicketTransaction tables
  • Handle email sending failures gracefully
  • Log all email attempts (success and failure)

Task 5.3: Create PromotionEmailSentLogService

Priority: MEDIUM

Description: Implement service for querying email logs

Required Methods:

  • listLogs(criteria, tenantId) - List logs with filtering
  • getLog(id, tenantId) - Get log entry by ID
  • getEmailStatistics(eventId, tenantId) - Get email statistics (optional)

5. REST API Controllers

5.1 PromotionEmailTemplateController

Task 6.1: Implement Template CRUD Endpoints

Priority: HIGH

Endpoints:

Method Path Description Response
GET /api/promotion-email-templates List templates (with filtering) PromotionEmailTemplateDTO[]
GET /api/promotion-email-templates/{id} Get template by ID PromotionEmailTemplateDTO
POST /api/promotion-email-templates Create new template PromotionEmailTemplateDTO
PATCH /api/promotion-email-templates/{id} Update template PromotionEmailTemplateDTO
DELETE /api/promotion-email-templates/{id} Delete template (soft delete) 204 No Content

Query Parameters (for GET list):

  • eventId.equals - Filter by event ID
  • isActive.equals - Filter by active status
  • sort - Sort order (e.g., "createdAt,desc")
  • page - Page number (0-indexed)
  • size - Page size

Requirements:

  • All endpoints must validate tenant ID from context
  • Use JHipster criteria API for filtering
  • Return appropriate HTTP status codes
  • Handle validation errors with clear messages

Task 6.2: Implement Email Sending Endpoints

Priority: HIGH

Endpoints:

Method Path Description Request Body Response
POST /api/promotion-email-templates/{id}/send-test Send test email { recipientEmail: string } { success: boolean, messageId?: string }
POST /api/promotion-email-templates/{id}/send-bulk Send bulk email { recipientEmails?: string[] } { success: boolean, sentCount: number, failedCount: number }

Requirements:

  • Test email: Send to single recipient (admin email)
  • Bulk email: Send to all event registrants if recipientEmails not provided
  • Log all email attempts in promotion_email_sent_log
  • Handle email failures gracefully
  • Return detailed success/failure counts

5.2 PromotionEmailSentLogController

Task 7.1: Implement Log Query Endpoints

Priority: MEDIUM

Endpoints:

Method Path Description Response
GET /api/promotion-email-sent-logs List sent email logs PromotionEmailSentLogDTO[]
